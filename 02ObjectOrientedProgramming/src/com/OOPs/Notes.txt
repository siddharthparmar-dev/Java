 JAVA RUNTIME DATA AREAS (JVM MEMORY)

 Java divides memory into different runtime areas for execution.
 These are classified into:
 1) Thread-specific areas
 2) Shared areas


 -------------------------------
 1. THREAD-SPECIFIC DATA AREAS
 -------------------------------

 a) Program Counter (PC) Register
 - Stores address of current executing instruction
 - Each thread has its own PC register
 - Undefined for native methods

 b) Java Stack
 - Stores stack frames (method calls)
 - Each method call creates a new frame
 - Each frame contains:
   * Local variables
   * Operand stack
   * Method return information

 Example:
 void fun() {
     int a = 10;   // stored in stack
 }

 Errors:
 - StackOverflowError (too many method calls)
 - OutOfMemoryError (stack overflow)

 c) Native Method Stack
 - Used for native (C/C++) methods
 - Similar to Java Stack


 -------------------------------
 2. SHARED DATA AREAS
 -------------------------------

 a) Heap Area
 - Stores objects and instance variables
 - Shared among all threads
 - Managed by Garbage Collector

 Example:
 Student s = new Student();  // object stored in heap

 Error:
 - OutOfMemoryError

 b) Method Area (Metaspace)
 - Stores:
   * Class metadata
   * Static variables
   * Method code
   * Runtime constant pool

 Example:
 static int count = 0;  // stored in method area

 Note:
 - In Java 8+, Method Area is called Metaspace

 c) Runtime Constant Pool
 - Part of Method Area
 - Stores:
   * String literals
   * Numeric constants
   * Class & method references

 Example:
 String s = "Hello";  // "Hello" stored in constant pool


 -------------------------------
 SUMMARY
 -------------------------------

 PC Register          -> Current instruction (Thread-specific)
 Java Stack           -> Method execution (Thread-specific)
 Native Method Stack  -> Native methods (Thread-specific)
 Heap                 -> Objects (Shared)
 Method Area          -> Class data (Shared)
 Constant Pool        -> Constants (Shared)


 -------------------------------
 QUICK REVISION
 -------------------------------

 Stack  = Method calls
 Heap   = Objects
 Method Area = Class-level data
 PC Register = Current instruction
 Constant Pool = Literals & constants

===> local variable : whenever a variable is declared inside a class. It is called local variable
