A Java program normally works using one thread.

A thread is simply:

“A path of execution”
or
“A worker doing a task”

Think of threads like workers in a restaurant.

One worker taking orders
One worker cooking
One worker cleaning

All workers can work at the same time.

That is multithreading.

In your code:

a.registration();
b.courseInfo();
g.printingStars();

these methods are running one after another because you are NOT using threads yet.

So execution flow is:

Registration completes
Then courseInfo starts
Then printingStars startsple threads simultaneously to perform multiple tasks efficiently
